Here's a few of my past and present projects.

This website

This site is running on a Linode Nanode instance running Ubuntu, with a domain name from NameCheap . The site is using Flask as a backend framework, with Apache acting as the HTTP server.

This particular stack was chosen due to my own personal prior experience in working with Python and Ubuntu. Particular emphasis is placed on keeping things as lean as possible; with only a single CPU core and 1GB of RAM available to my particular cloud instance price plan, I wanted to run a relatively lightweight setup that could leave me with enough resources to tinker with in the background.

DV Stats: A server status monitoring app

The inspiration for an app that could regularly ping my server and alert me in case of downtime arose when I tried to remote into my server instance and saw my connection timed out. After a few panicked minutes, including double-checking my DNS records were pointed properly, I received an email from my hosting server provider informing me that they had been made aware of a physical hardware incident with the server my instance occupied.

I determined then that I needed some sort of at-a-glance way of seeing that things were still running properly, and ideally a way to alert me without having to check in myself.

The app is still in early development, but can already ping a hardcoded API endpoint set up through my Flask backend through the Fuel networking library. Some basic functionality still on to to-do list:

  • Formatting received API responses (JSON parsing)
  • Allowing for user-input API call locations, and storing (multiple) locations to regularly connect to
  • Most of the UI; from design to implementation/how should the app look and function/etc
  • Notifications and alerts if server does not respond/responds with e.g high CPU usage value

The app is also written in the exciting new Jetpack Compose system recently fully released by Google, which brings some interesting new paradigms to UI development in the Android world.
